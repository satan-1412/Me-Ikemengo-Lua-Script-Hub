[ ⚠️ CRITICAL REMINDER: CNS State File Mounting ]
Although the current Lua script supports automatic directory recognition, the Z-axis common state file (z_axis_auto.cns) generated by the script still needs to be declared in the engine. Otherwise, states 85200 and 85201 cannot take over the character!

Operation Path:
1. Open save/config.ini in the game's root directory (some repacks use data/config.ini).
2. Locate the configuration section that controls Common States.
3. Add save/z_axis_auto.cns to the reading list of common states.

--------------------------------------------------------------------
[ I. Lua Script Installation (Auto-Recognition Mechanism) ]
1. Save the Lua code as a .lua file with any name (e.g., auto_z_system.lua).
2. Place this file into the external/ or modules/ folder in the game's root directory.
3. The engine will automatically recognize and load the script upon startup; no manual code mounting is required.

--------------------------------------------------------------------
[ II. Script Internal Configuration (SYSTEM_CONFIG Area) ]
Open your Lua script with a text editor and modify the SYSTEM_CONFIG variables at the top as needed:

> Basic Automation Switches:
ENABLE_AUTOLOAD_CHARS  = true  ; [span_1](start_span)Enable automatic reading and loading of characters[span_1](end_span)
ENABLE_AUTOLOAD_STAGES = true  ; Enable automatic reading and loading of stages
ENABLE_AUTO_MOVELIST   = true  ; [span_2](start_span)Enable automatic parsing of .cmd to generate movelists[span_2](end_span)

> Z-Axis Movement System Settings:
ENABLE_Z_AXIS = true           ; [span_3](start_span)Master switch to enable Z-axis depth movement in 3D stages[span_3](end_span)
Z_MOVE_SPEED  = "2.2"          ; [span_4](start_span)Depth sliding speed (Recommended: 1.5 - 3.5)[span_4](end_span)

> Operation Mode Selection (USE_UP_DOWN_FOR_Z_MOVE):
- Set to true (Classic Mode): The ↑/↓ arrow keys become inward/outward Z-axis movement. Requires configuring a spare jump key (default: w) and a spare crouch key (default: hold d).
- Set to false (Split Mode): The ↑/↓ arrow keys retain native jump/crouch. Requires configuring a Z-axis inward key (default: hold w) and a Z-axis outward key (default: hold d).

--------------------------------------------------------------------
[ III. Key Mapping Configuration (Modify common.cmd) ]
Regardless of which operation mode you choose above, you must register the corresponding trigger commands in data/common.cmd for the system to receive key signals.

Instructions:
Open data/common.cmd, and directly append the following code just below the [Defaults] tag near the top of the file (DO NOT overwrite any existing code):

; -[span_5](start_span)-- Z-Axis System Custom Key Mapping ---[span_5](end_span)
[Command]
name = "w"          
command = w         
time = 1            
buffer.time = 1     
buffer.hitpause = 1 
buffer.pauseend = 1 

[Command]
name = "hold_w"     
command = /w        
time = 1
buffer.time = 1
buffer.hitpause = 1
buffer.pauseend = 1

[Command]
name = "hold_d"     
command = /d
time = 1
buffer.time = 1
buffer.hitpause = 1
buffer.pauseend = 1
